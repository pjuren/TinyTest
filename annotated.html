TestCaseThis class is the base test case class. User defined test cases will extend this class (though they won't know it necessarily, since it's done automatically for them using a macro)
TestCaseAdderThis is a helper class that facilitates automatic registration of tests. When one of these is constructed, it is given a TestSet pointer (in reality, a pointer to an object that is a subclass of TestSet), which it registers with a static test set maintained by the TestSet class
TestSetThis class defines a Test Set, which is really just a collection of test cases. Uses the factory pattern; objects of this class should not be directly constructed. When one is needed, call getTestSet()
TinyTestExceptionExceptions of this type are thrown whenever a test fails
